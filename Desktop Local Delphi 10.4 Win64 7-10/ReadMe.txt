Представляю Вашему вниманию Pre-Desktop local Win 64, написанную на Delphi 10.4 Sidney. На версиях 32 bit не работает версия TensorFlow для Python 3.8.9.
Необходимо использовать более ранюю версию - очевидно, Python 3.7. Более ранние версии не поддерживаются.

Предварительно проверте все пути, указанные в настройках.

============================================================================

Если не загружает модуль keras и tensorflow - необходимо апгрейдить драйвер видеокарты и 
установить последнюю версию - Redistributable Visual C++. Поставил и запустил на видеокартах (разные системники) 
NVIDIA GeForce GTX 645 4 GB - Win7 64 - не инсталлированы Python и Delphi и
NVIDIA GeForce GT 730 4 GB - Win7 64, Win10 64. На системы Win32 не ставится tensorflow.

============================================================================

Для доступа к Python 3.8 используются следующие компоненты и библиотеки:
Python4Delphi
PythonEnviroments
Lightweight-Python-Wrappers
PythonPackages4Delphi
P4D-Data-Sciences.
Для интерфейса использовались частично
rxlib_275_unof_1_0_21.

Структура файлов:
Папка программы:
\Project Seg Delphi 10.4 (Win10 64 или Win 7.0 64)
- \Images - Папка с файлами с входной, Resize и выходной картинкой - все пути можно переопределить в программе
- \Model - Папка с файлом модели
--\Win64\Release - Папка с исполняемым файлом и необходимыми библиотеками
---\3.8 - Папка с Embeded версией Python 3.8.9. 

По факту .exe должен сам скачивать необходимые библиотеки из Интернета, но по факту не всегда скачивает все, что нужно, не знаю почему. 
Виснет на скачивании TensorFlow, хотя раза два скачал - от чего зависит, не понял. Не скачиваются пакеты полностью (зависимости). К примеру, при чистой установке, не скачивается PIL.Image. Взял все пакеты из инсталляции Python 3.8.9. Находятся в папке 3.8 - тогда вроде работает.

По итогу используются библиотеки keras-2.12.0., NumPynumpy-1.23.5. и Pillow-9.5.0. (PIL). По факту еще нужен tensorflow-2.12.0. с сопутствующими взаимосвязанными библиотеками (packages).
Требует также апгрейдить pip до версии pip-23.0.1.

Выход: Файл jpg с размерами для подачи в Mодель и, собственно, результат работы Mодели.

Зы. Пока распознается одна картинка - по факту будем обрабатывать пул картинок, потом "склеивать" их для приведения к входной картинке - перед этим разрезаем исходную картинку на размеры, с которыми будет работать модель.
При загрузке проявите терпение - packages инсталлируются и импортируются, также в память загружается сам Python 3.8.
Приветствуются замечания и пожелания по интерфейсу. Значения path и др. редактируются и сохраняются в файл настроек Properties.ini (в папке, где лежит исполняемый файл).

Кому интересно, исходники могу загрузить.

Зы. Зы. Если кто-то хочет использовать свою модель сегментации - Укажите пути к модели и картинке на первой вкладке, а последней вкладке - размеры картинки, которую подаете в модель, цветность (1 или 3) 
и количество классов цветов сегментирования. Массив классов пока не вынес отдельно - сделаю, для настройки отображения сегментированной картинки. Поэтому количество классов пока 0-6 и они пока предустановленные.
